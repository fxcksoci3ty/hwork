1. Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?

ОТВЕТ: git checkout <имя-файла>

2. Как вернуть файл из индекса в рабочий каталог?

ОТВЕТ: git reset HEAD <имя-файла>

3. Как отменить ошибочный commit?

ОТВЕТ: git reset HEAD~<количество коммитов>

4. Как исправить коммит отправленный на сервер GitHub?

ОТВЕТ: git commit --amend 

5. Как создать новую ветку и переключится в нее?

ОТВЕТ: git checkout -b <имя-ветки>

6. В чем отличение между fast-forward merge and non fast-forfard merge?

ОТВЕТ: Fast-Forward Merge происходит, когда у нас нет новых коммитов в той ветке, которую мы сливаем. Также не создаётся коммит слияния, ищменения просто добавляются к текущей ветке. Non Fast-Forward Merge происходит, когда история изменения ветки "не линейна" и когда ветка, в которую мы сливаем, имеет коммиты, после того, как была создана из текущей ветки.

7. Когда может возникнуть конфликт слияния веток, как его разрешить?

ОТВЕТ: Конфликт слияния происходит, когда при слиянии с двух веток, изменения содержатся в одной и той же части одного (или нескольких) файла(-ов). Чтобы разрешить конфликт, нужно просмотреть файлы, где есть конфликты и принять решение, что оставить, удалить маркеры конфликта (<<<<<<, =======, >>>>>>>), добавить измененные файлы с помощью git add и сделать коммит.

8. Можно ли удалить ветку? Как? 

ОТВЕТ: git branch -d <имя-ветки>
